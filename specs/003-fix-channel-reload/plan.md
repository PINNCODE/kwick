# Implementation Plan: Fix Channel Reload on Menu Open/Close

**Branch**: `main` | **Date**: 2026-05-10 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `specs/003-fix-channel-reload/spec.md`

**User Requirements**:
1. Fix channel reload issue when opening/closing menu
2. Implement two-step cascade menu: Categories only → then Channels+EPG together
3. Back button returns to categories-only view

---

## Summary

**Primary Goal**: Fix the UX issues with the cascade menu: (1) prevent channel reload when toggling menu, and (2) implement a cleaner two-step flow where categories are shown first, then channels+EPG appear together after selection.

**Technical Approach**:
- Add `viewMode` state to `useCascadingMenu` hook ('categories' | 'channels')
- Separate panel visibility (`viewMode`) from panel focus (`activePanel`)
- Conditionally render ChannelsPanel + EPGPanel based on `viewMode`
- Modify back button to switch viewMode instead of just changing panel focus
- Reset viewMode to 'categories' on menu open/close

### Key Research Decisions

   - Rationale: Easy to understand, test, and extend

5. **Back Button**: Returns to categories-only view (hides channels+EPG)

6. **State Reset**: Always reset to categories view on menu close/open
